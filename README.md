
# âš¡ MeetupOS

**"We gather moments, not just people."**

MeetupOS is a real-time, mobile-first companion web application designed to gamify offline meetups. It replaces boring presentations with interactive games, live voting, and anonymous confessions, all controlled from a central Admin Dashboard.

## ğŸš€ Features

### For the Admin (RC/Host)

  * **Command Center:** A desktop-optimized dashboard to control the entire room.
  * **Live Player List:** See who is online in real-time.
  * **Moderation Tools:** "Kick" unwanted users or "Buzz" (vibrate) a student's phone to get attention.
  * **Game Control:** seamless switching between game modes.
  * **Question Manager:** Bulk upload Truths, Dares, and Trivia via JSON file.

### For Students (Mobile)

  * **No Login Required:** Automatic guest authentication (one device = one account).
  * **Real-Time Sync:** Screens update instantly when the Admin changes the game.
  * **Haptic Feedback:** Phones vibrate when buzzed or when it's their turn.
  * **Interactive UI:** Neo-brutalism design style for a fun, casual vibe.

### ğŸ® The Games

1.  **RNG (Truth or Dare):**
      * Anti-repeat logic (remembers recent victims).
      * "The Tribunal": Trigger live Pass/Fail voting from the audience.
2.  **Over/Under (Tournament Mode):**
      * Best of 3 or 5 rounds.
      * Automated scoring and Grand Champion calculation.
3.  **Confessions:**
      * Themed prompts (e.g., "Hostel Horror Stories").
      * Admin curates and "Broadcasts" the best ones to everyone's screen.

## âš™ï¸ Setup & Installation

### 1\. Clone the Repository

```bash
git clone https://github.com/yourusername/meetup-os.git
cd meetup-os
```

### 2\. Configure Supabase

1.  Create a new project at [Supabase](https://supabase.com).
2.  Go to the **SQL Editor** in your dashboard.
3.  Run the following SQL script to set up the tables and Realtime permissions:

<!-- end list -->

```sql
-- Create Tables
create table rooms (
  id text primary key,
  mode text default 'LOBBY',
  game_state jsonb default '{}'::jsonb, 
  confess_theme text default 'General',
  broadcast_msg text,
  tribunal_active boolean default false,
  updated_at timestamptz default now()
);

create table users (
  uid text primary key,
  name text,
  is_kicked boolean default false,
  is_vibrate boolean default false,
  last_seen timestamptz default now()
);

create table tasks (
  id bigint generated by default as identity primary key,
  type text,
  text text
);

create table questions_ou (
  id bigint generated by default as identity primary key,
  q text,
  a text
);

create table confessions (
  id bigint generated by default as identity primary key,
  text text,
  theme text,
  created_at timestamptz default now()
);

create table guesses (
  id bigint generated by default as identity primary key,
  uid text,
  name text,
  guess int,
  created_at timestamptz default now()
);

create table votes (
  id bigint generated by default as identity primary key,
  vote text,
  created_at timestamptz default now()
);

-- Enable Realtime
alter publication supabase_realtime add table rooms;
alter publication supabase_realtime add table users;
alter publication supabase_realtime add table confessions;
alter publication supabase_realtime add table votes;

-- Init Room
insert into rooms (id, mode) values ('KANPUR25', 'LOBBY');
```

### 3\. Connect the App

1.  Open `app.js`.
2.  Find the config section at the top.
3.  Paste your **Project URL** and **Anon Key** from Supabase Settings \> API.

## ğŸ“‚ How to Add Questions (Bulk Upload)

You don't need to enter questions manually. Create a `data.json` file on your computer with this structure:

```json
{
  "truths": [
    "What is your biggest fear?",
    "Who is your crush?"
  ],
  "dares": [
    "Do 10 pushups.",
    "Dance like a chicken."
  ],
  "ou": [
    {"q": "How many stairs to library?", "a": "140"},
    {"q": "Age of IITK?", "a": "64"}
  ]
}
```

1.  Open the **Admin Dashboard** (Login with `1234`).
2.  Click **ğŸ“‚ Upload Questions**.
3.  Select your JSON file.
4.  Done\! The database is now seeded.
